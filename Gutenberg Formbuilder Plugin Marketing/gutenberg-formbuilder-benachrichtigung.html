E-Mail-Benachrichtigung · Gutenberg Formbuilder

Eine Anfrage kommt rein –
die richtige Person weiss es sofort.

Wer täglich ins Backend schaut, verpasst Anfragen oder antwortet zu spät. Gleichzeitig sollen sensible Inhalte nicht ungeschützt per Mail herumgehen. Die Lösung: E-Mail-Benachrichtigung direkt aus WordPress – kontrolliert, sicher, ohne Umweg.

Das Problem ohne Benachrichtigung
  • Anfragen warten stundenlang, weil niemand regelmässig ins Backend schaut.
  • Vertrauliche Informationen landen im Klartext in einem weitergeleiteten Mail.
  • Drittanbieter-Webhooks (Zapier, Make) bedeuten eine weitere Datenweitergabe.
1 – Wie es funktioniert

Fünf Einstellungen – eine Benachrichtigung.

Die E-Mail-Benachrichtigung konfigurieren Sie direkt im Gutenberg-Inspector des Formular-Blocks. Neu in Version 2.5.0 – standardmässig ausgeschaltet, pro Formular aktivierbar.

1

Pro Formular einschaltbar

Jedes Formular hat seinen eigenen Schalter. Standardmässig ist die Benachrichtigung deaktiviert – Sie schalten sie nur dort ein, wo es Sinn ergibt.

✓ Kein «alles oder nichts» – Kontaktformular ja, internes Erfassungsformular nein
2

Empfänger frei wählbar

Tragen Sie eine oder mehrere E-Mail-Adressen ein (kommagetrennt). Leer lassen Sie das Feld nicht – dann greift der Fallback auf die WordPress-Admin-Adresse.

Beispiel: sekretariat@verein.ch, vorstand@verein.ch
3

Betreff mit Platzhaltern

Der Betreff der Benachrichtigung kann Feldwerte aus der Einsendung enthalten. Den technischen Feldnamen sehen Sie im Inspector unter jedem Feld.

Beispiel: «Neue Anfrage von {{vorname}} {{nachname}}»
4

Absender wählbar

Drei Optionen: die WordPress-Admin-Adresse (Standard), eine feste eigene Absenderadresse, oder ein E-Mail-Feld der Einsendung – so geht «Antworten» direkt an die anfragende Person.

✓ Eigener Absender seit Version 2.4.1
5

Sicher im Inhalt

Verschlüsselte Felder und Datei-Anhänge erscheinen niemals im Klartext in der Benachrichtigung. Stattdessen steht im Mail ein Hinweis: «Dieser Eintrag enthält vertrauliche Daten – bitte im Backend ansehen.»

⚠ Das ist Absicht: Wer eine Benachrichtigung breiter verteilt, gibt keine sensiblen Daten weiter.

2 – Was Sie davon haben

Weniger warten, mehr Kontrolle.

Die Benachrichtigung ist kein Extra-Tool. Sie läuft im Hintergrund Ihres bestehenden WordPress – ohne neuen Login, ohne neuen Anbieter.

📬

Anfragen landen sofort beim richtigen Menschen

Die zuständige Person sieht die Anfrage im selben Moment, in dem sie eingeht – im Postfach, das sie sowieso offen hat. Kein täglicher Backend-Check nötig.

🏠

Mail läuft über Ihr eigenes WordPress

Kein Drittanbieter-Postfach, kein Zapier-Zwischenschritt, kein Make-Workflow. Das Plugin nutzt wp_mail() – die Mail geht über die Infrastruktur Ihres Hostings, nicht über einen Fremden.

🔒

Sensible Daten bleiben geschützt

Selbst wenn die Benachrichtigung an fünf Adressen geht: Verschlüsselte Felder und Datei-Anhänge erscheinen nie im Klartext in der Mail. Wer den Eintrag vollständig sehen will, muss ins Backend – mit der richtigen Berechtigung.

↩

Direkt antworten, ohne suchen

Wenn der Absender auf das E-Mail-Feld der Einsendung gesetzt ist, genügt ein Klick auf «Antworten» im Mail-Client – die Antwort geht direkt an die anfragende Person. Kein Kopieren von Adressen aus dem Backend.

3 – Was wir nicht versprechen

Ehrlich und vollständig.

Drei Punkte, die Sie kennen sollten – damit die Benachrichtigung das hält, was Sie erwarten.

Kein Spam-Filter-Bypass

Das Plugin schickt die Mail über wp_mail(). Ob sie im Posteingang oder im Spam-Ordner landet, hängt von der Konfiguration Ihres Hostings ab – insbesondere von SPF, DKIM und DMARC. Das Plugin beeinflusst das nicht. Bei Problemen: SMTP-Plugin einrichten oder Hosting kontaktieren.

Kein Bestätigungsmail an Absender

Das Plugin schickt keine automatische Bestätigung an die Person, die das Formular ausgefüllt hat. Diese Funktion birgt datenschutzrechtliche Fragen (Einwilligung, Doppelklick-Opt-in), die je nach Kontext unterschiedlich beantwortet werden. Wer eine Bestätigung braucht, löst das heute über den Erfolgsbereich oder eine eigene Danke-Seite.

Keine SMTP-Konfiguration enthalten

Das Plugin richtet keinen eigenen SMTP-Ausgang ein. Es nutzt, was WordPress bereitstellt – wp_mail() und das Hosting-Setup dahinter. Wer einen eigenen SMTP-Dienst (Mailgun, Postmark, SendGrid) einrichten möchte, tut das über ein SMTP-Plugin seiner Wahl; das funktioniert nahtlos.
