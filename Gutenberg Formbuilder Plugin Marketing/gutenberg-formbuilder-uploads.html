Datei-Uploads sind die häufigste Angriffsfläche von Formularen. Falsche MIME-Typen, getarnte Endungen, Pfade ausserhalb des erlaubten Bereichs. Und wenn etwas Bösartiges durchkommt, liegt es in wp-content/uploads/ und ist von aussen abrufbar.

1 – Wie es funktioniert

Vier Schutzebenen – automatisch, ohne Konfiguration im Alltag.

Jede hochgeladene Datei durchläuft alle Prüfungen bevor sie gespeichert wird. Fällt eine Prüfung durch, landet die Datei nirgendwo.

1

Privates Verzeichnis – kein öffentlicher Pfad

Dateien werden niemals in wp-content/uploads/ gespeichert. Stattdessen legt das Plugin beim ersten Start das Verzeichnis wp-content/.gfb-private/ an, mit Zugriffsrechten 0600. Kein direkter URL-Aufruf ist möglich – weder für Besucherinnen noch für Suchmaschinen, auch wenn die Apache- oder Nginx-Konfiguration suboptimal ist.

✓ Eine .htaccess-Vorlage und eine Nginx-Direktive werden mitgeliefert – als doppelter Schutz.
2

MIME-Whitelist und echte Dateityp-Erkennung

Das Plugin prüft nicht nur die Dateiendung – es liest die tatsächliche Datei-Signatur via finfo. Eine Datei, die als bericht.pdf hochgeladen wird, aber intern eine ausführbare Datei ist, fliegt heraus. Doppelte Endungen wie anhang.pdf.exe werden blockiert. Erlaubte Typen (PDF, JPG, PNG, DOCX, XLSX …) sind in der Plugin-Konfiguration als Whitelist definiert – was nicht explizit erlaubt ist, ist verboten.

✓ Pro Formular-Feld konfigurierbar: Welche Typen sind erlaubt, wie gross darf die Datei maximal sein?
3

ClamAV-Virenscan – optional als Pflicht, optional als Fallback

Wenn Ihr Server ClamAV unterstützt, scannt das Plugin jede hochgeladene Datei vor der Speicherung. Drei Modi stehen zur Wahl:

clamscan clamdscan unix-socket

In den Plugin-Einstellungen wählen Sie: Soll ClamAV zwingend verfügbar sein (Upload schlägt fehl, wenn der Scanner nicht antwortet), oder soll er stillschweigend übersprungen werden, wenn er nicht erreichbar ist? Ein EICAR-Selbsttest ist mitgeliefert – so prüfen Sie nach der Einrichtung, ob der Scanner korrekt antwortet und Testdateien zuverlässig erkennt.

✓ ClamAV ist optional – ohne Scanner läuft das Plugin mit allen anderen Schutzmassnahmen uneingeschränkt.
4

Gesicherter Download-Endpoint mit Berechtigungsprüfung

Datei-Anhänge können nur über einen kontrollierten WordPress-Endpoint abgerufen werden. Der Endpoint prüft die Berechtigung (gfb_download_files), setzt den Header application/octet-stream, damit Browser die Datei speichern statt öffnen, und schreibt für jeden Download einen Eintrag ins Audit-Log mit Zeitstempel und Benutzerin. Kein direkter Dateipfad, kein direkter Zugriff über den Webserver.

✓ Jeder Download ist nachvollziehbar – wer, wann, welche Datei.
2 – Was Sie nicht mehr tun müssen

Weniger manuelle Konfiguration. Weniger Angriffsfläche.

Diese Massnahmen müssen bei vielen WordPress-Plugins manuell eingerichtet werden. Der Gutenberg Formbuilder liefert sie standardmässig mit.

automatisch mitgeliefert
🔒

Apache & Nginx absichern

Wer ein privates Upload-Verzeichnis betreibt, muss normalerweise den Webserver sperren. Das Plugin legt beim ersten Start eine .htaccess-Datei im privaten Verzeichnis ab und liefert eine kommentierte Nginx-Vorlage im INSTALL.md. Beides ist fertig – Sie kopieren die Nginx-Direktive, das war's.

automatisch
🎲

Sichere Dateinamen

Originaldateinamen werden nicht gespeichert – das Plugin vergibt beim Upload eine zufällige ID als internen Dateinamen. Kein Pfad-Traversal, keine Kollisionen, keine Rückschlüsse auf den Inhalt anhand des Namens. Den Originalnamen kennt nur die Datenbank, und er wird beim kontrollierten Download korrekt gesetzt.

Audit-Log
📋

Nachvollziehen, wer was heruntergeladen hat

Wer hat welchen Anhang wann heruntergeladen? Ohne Audit-Log müssten Sie Serverprotokolle durchsuchen – und dort fehlen Kontext und Benutzerzuordnung. Das Plugin schreibt jeden Datei-Download automatisch ins Audit-Log: Benutzerin, Zeitstempel, Datei-ID, Einsendungs-ID. Manipulationssicher durch verkettete Prüfsummen.

3 – ZIP-Export der Anhänge

Alle Dateien geordnet – auf einen Klick.

Formulare mit Datei-Uploads bieten im Backend nicht nur den CSV-Export, sondern auch einen vollständigen ZIP-Export.

📦

Was der ZIP enthält

Der ZIP enthält die CSV-Datei aller Einsendungen plus alle hochgeladenen Dateien. Jede Einsendung bekommt einen eigenen Unterordner – der Ordnername ist die Absender-E-Mail-Adresse. So sind Datei und Eintrag ohne Suchen einander zugeordnet: bewerbungen.zip / anna.mueller@mail.ch / lebenslauf.pdf. Verschlüsselte Datei-Anhänge bleiben im Export verschlüsselt; der Klartext-Download setzt die Entschlüsselungs-Berechtigung voraus und wird im Audit-Log protokolliert. Verfügbar seit Version 2.5.0.

Mehr zum Export-Bereich: Seite «Export & CSV».

4 – Was wir nicht versprechen

Ehrlich über die Grenzen.

Diese Funktionen sind nicht enthalten. Wer sie braucht, findet sie anderswo.

Eigener Antivirus-Dienst

Das Plugin integriert ClamAV – aber ClamAV muss auf dem Server installiert sein. Auf Shared-Hosting-Umgebungen ist das selten der Fall. Wer keinen ClamAV-Zugang hat, kann den Upload-Schutz ohne Scanner betreiben – alle anderen Massnahmen (MIME-Prüfung, privates Verzeichnis, Berechtigungs-Endpoint) bleiben wirksam.

Bildverarbeitung und Thumbnail-Generierung

Hochgeladene Bilder werden nicht skaliert, beschnitten oder als Vorschau angezeigt. Das Plugin nimmt die Datei entgegen, prüft sie und speichert sie sicher – mehr nicht. Wer Vorschaubilder in der Einsendungsübersicht braucht, ist mit einem anderen Werkzeug besser bedient.
